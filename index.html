Dive into HTML

The content layer

Good file naming habits

To create an HTML web page, just name your file using the .html extension (much like .pdf for PDF files or .doc for Word files).

  • Use lowercase letters.
    about.html instead of About.html or ABOUT.html
  • Avoid s p a c e s. Use underscores (_) or dashes (-). Dashes are generally preferred for SEO (search engine optimization).
    business_hours.html or business-hours.html instead of business hours.html
    (The browser will interpret it as business%20hours.html)
  • Avoid symbols like # & * in your file name.
  • Keep your file names concise, yet meaningful.
    about.html, contact.html instead of page1.html, page2.html

These file naming tips can be applied to any files such as images, folders, CSS, etc.

What is HTML?

HTML stands for HyperText Markup Language. It was created for web browsers to easily read.

Below are the basic HTML tags required for all web pages. This makes perfect sense to a web browser:

<!DOCTYPE html><html><head><title></title></head><body></body></html>

Just like the different languages that we speak and write, HTML has to follow proper "grammar" rules as well to make sense.

This is referred to as syntax.

What is HTML?

Whitespace through indentation (tabs) make your HTML more human-readable.

<!DOCTYPE html>
<html>
    <head>
        <title> </title>
    </head>
    <body>
    </body>

HTML tags

The structure of a web page is defined by HTML tags. Left & right angled brackets (shift + < and shift + > ) surround the tag name like this:

<tag>

Think about HTML tags like the beams of a house. It's what holds up a web page can creates structure.

HTML tags

Tags typically come in pairs — an opening tag and a closing tag with a forward slash:

<tag></tag>

The are many different HTML tags. Each are used to create structure and add semantics (meaning) to the content.

For example, any content between the <html></html> tag tells the browser that it is HTML code.

Content between the <title></title> tag describes the web page title.

HTML tags: nesting

Tags can also be nested within other tags.

<tag><tag></tag></tag>

Let's look at the HTML tags that are required for a basic web page and take a closer look at how they are nested.

HTML tags: nesting

Nested elements make up the document tree & the relationships are just like a family tree; branches of ancestors, parents, children, siblings, etc.

<!DOCTYPE html>
<html>
    <head><!-- child of html tag, parent of title tag -->
        <title></title>
    </head>
    <body><!-- child of html tag -->
    </body>
</html>

    All other HTML tags are nested within <html>.
  • <head> is the child of <html> and the parent node of the <title> tag.

HTML Document Type

What about the 1st line?

<!DOCTYPE html>

This is a Document Type Declaration a.k.a DOCTYPE and must always be included as the first line of the document.

This tells the web browser what version of HTML the document is written in and helps the browser render the content based on the rules specific to that version of HTML.

The W3C (World Wide Web Consortium) governs web standards with specifications, guidelines, tools and recommendations. They also provides a free service to check if your markup is valid: http://validator.w3.org/

HTML, XHTML, HTML5

These terms refer to different versions of the same language though the rules, syntax and features may differ. However, the HTML5 DOCTYPE is backwards compatible and can be used in place of the older versions.

Consider how old versions of MS Word or Photoshop have problems opening files created in later versions of the software.

HTML5 DOCTYPE

<!DOCTYPE html>

XHTML DOCTYPE

<!DOCTYPE html PUBLIC "-//W3C//DTD XHTML 1.1//EN" 
"http://www.w3.org/TR/xhtml11/DTD/xhtml11.dtd">

HTML4 DOCTYPE

<!DOCTYPE HTML PUBLIC "-//W3C//DTD HTML 4.01//EN"
"http://www.w3.org/TR/html4/strict.dtd">

HTML content: ADDING TEXT

Heading tags are used for headings/page titles.

There are six levels of ranging from <h1> to <h6>. They follow a hierarchy with <h1> being the most significant heading on the page.

<h1>Name of blog</h1>
<h2>Title of blog post</h2>
<h3>Sub headings within blog post</h3>

Paragraphs are represented by the <p> tag.

<p>This is a paragraph.</p>
<p>This is another paragraph.</p>

<b> or <strong>, <i> or <em>

In XHTML, <b> and <i> were no longer used. In HTML5, they were re-introduced with a slightly different meaning/usage. See here for more info.

To keep from forgetting to close your tags, it's good practice to write the open & close tags first, then go back and add the content between the tags.

HTML LISTS

Ordered lists are for items that need to follow a specific order. Two HTML tags are required, an ordered list tag <ol> and a list item <li> tag.

<ol>
    <li>Buy ticket to LLC event.</li>
    <li>Arrive at venue.</li>
    <li>Turn on your computer.</li>
    <li>Learn some HTML & CSS!</li>
</ol>

Unordered lists are for items that don't follow any order and use the unordered list tag <ul> instead.

<ul>
    <li>Breaking Bad</li>
    <li>Game of Thrones</li>
    <li>Arrested Development</li>
    <li>The Wire</li>
    <li>True Blood</li>
</ul>

HTML tags and other content can only be added between the <li> tags, not the <ul> or <ol> tags.

Void (or empty) HTML Tags

Void tags don't format content — instead, they render or embed objects in our page. These tags do not come in pairs like h1 or p tags.

However, In the XHTML versions (remember the different DOCTYPE rules?) they self-close with a forward slash (/) immediately before the right angled bracket (>).

In HTML5, void tags are not required to self close but also accepts self-closed void tags as well.

A common void tag is the <br /> or <br> tag used to create a line break.

<p>This text<br />goes over<br />several<br />lines.</p>

Attributes

Attributes are used to render special content like hyperlinks, images, or video because they provide additional information or instruction.

Here's that same img tag but now it's using an attribute (src), to tell the web page where to find the source of the image.

Remember, img is a void HTML tag so it does not need a closing tag. This is wrong: <img>http://goo.gl/7eSU5</img>

Working with images

Images are an example of a void tag with an attribute to define the filepath of the image file.

<img src="http://goo.gl/7eSU5" />
  • src is the attribute
  • http://goo.gl/7eSU5 is the value of that attribute
    Values are typically surrounded by single or double quotes since occassionally values may contain spaces.

With an img tag, the value of the src attribute tells the browser what the url source of our image is.

Working with images

Tags can have more than one attribute. Some are required (src) and others are optional (width and height). These attributes tell the browser what dimension to display the image at.

(This is useful so the page doesn't jump around when loading very large images, or while on a slow internet connection such as dial-up or 2G/3G. Also, width/height values don't necessarily have to match the image's actual width/height.)

For example, without width and height attributes, the browser displays image at original size.

<img src="http://goo.gl/7eSU5" />

Now with a manipulated width and height (about 50% smaller):

<img src="http://goo.gl/7eSU5" width="320" height="262" />

Working with images

alt is another optional image attribute and refers to alternative text. It is used to display to visually impaired web visitors using screen readers.

It also shows up when there's a broken or missing image due to typos or moved files.

The img tag will work without the alt attribute but it's required to be "valid" HTML markup.

CSS basics

The presentation layer

What is CSS?

  • CSS = cascading style sheet
  • stylesheet language with its own syntax rules that are different from HTML
  • Separates content (i.e. text) from presentation (i.e. text colour).
  • a style sheet contains a list of rules that cascade and can help to reduce redundancy and save time

There are three ways to include (otherwise known as referencing)
CSS in an HTML page: inline, internal and external.

Inline CSS

The style attribute can be added to any visible HTML tag. This allows us to add CSS properties directly to the element.

Let's practice using using the background-color, color and
font-size property.

Internal CSS using selectors

Rather than using a style attribute directly in the element, a <style> tag is declared and the CSS declarations go in between these <style> tags.

Here's the same result but using CSS selectors.


NOTE: Internal CSS should be place between the <head> tags.

Common CSS terms

Selectors: used to determine which element(s) to apply the styles to. Can target an HTML tag (i.e. <h1> for a heading), class or ID.

Declaration blocks consists of a list of declarations (rules to follow) wrapped in curly braces { }.

Declarations are written using CSS property: value pairs.

selector { property: value; }

h1 { color: blue; }

CSS selectors and properties

Here's a CSS declaration dissected:

  • A basic selector will match its HTML tag name. In this case, all content that is in an <h1> will have this style applied to it.
    (We'll get into more complex ones later.)
  • A property is a label used to determine the type of style to be applied to the element (like color). You can see a list of them in this cheatsheet.
  • The property value is specific to the property type. For example, color properties accept colour values like blue or #FFFF00. font-size properties accept units of measurements like 12px (where "px" stands for pixels).

CSS selectors and properties

Here's a CSS declaration dissected:

  • Use curly braces ({}) to hold a list of the property:value sets.
  • You can define multiple property:value sets but you must end each one with a semi-colon (;).

Changing the font colour

Now the text is a bit hard to read on the gray background.

The <body> tag is the highest possible visible HTML tag, so the body selector will also allow us to target the font color on the entire page.

body { background-color: gray; color: white; }

Changing the font family

The web has a number of fonts that are considered "web safe". Some of the most used fonts are:

  • Arial
  • Verdana
  • Times New Roman
  • Georgia
  • Courier

Let's add a bit of contrast to our headers by using the Courier font.

h1 { font-family: Courier; }
h2 { font-family: Courier; }

Changing the font family

Not all fonts are installed on all computers so it's recommended that you use a list of fonts. If the first font in the list isn't available, it will try to use the next.

body { background-color: gray; color: white; font-family: Helvetica, Verdana, Arial, sans-serif; }

(Notice how the font-family for the headers are still Courier? That's the "cascading" part of Cascading Style Sheets in action. Inline styles take precedence, then innermost tags, then outer tags.)

Here's a handy resource, CSS Font Stack.

Changing the font size

Font sizes are typically defined in pixels (px) or "ems" (em).

  • 20px
  • 45px
  • 0.5em
  • 1em

Headers have default font-sizes defined by the browser. Let's override them. 

h1 { font-family: Courier; font-size: 25px; }
h2 { font-family: Courier; font-size: 22px; }

(When you've got more time, read this article for more info on font unit sizes.

Changing the font weight

In its simplest form, there are 2 weights:

  • normal
  • bold

By default, headers are already bolded. Let's remove the bold by applying the normal font weight on them.

h1 { font-family: Courier; font-size: 25px; font-weight: normal; }
h2 { font-family: Courier; font-size: 22px; font-weight: normal; }

CSS classes

If you know that several elements are going to have similiar styles, use CSS classes! A class attribute can be added to any element and given any value, defined by you. Once you define a CSS class, you can apply it over and over again, to any element.

  • Add a class attribute to the HTML element and give it a meaningful class name. e.g. class="nav-button"
  • In CSS, a class is defined by a leading period (.). Use the leading period, plus class name to define your CSS selector
    e.g. .nav-button 
<a class="nav-button">Home</a>

<style>
.nav-button {
	font-family: Georgia;
}
</style>

External CSS

Instead of duplicating your <style> content for every web page, make one single CSS document ending with a .css extension and link to it in the <head> tag instead of (or in addition to) a <style> tag.

<head>
	<link rel="stylesheet" href="styles.css" type="text/css" />
	<style>
	body {
		font-size: 14px;
	}
	</style>
</head>

Inside styles.css, there is no <style> tag:

body {
	background-color: black;
	color: white;
	font-family: Helvetica, Arial, sans-serif;
}
h1 {
	font-family: Georgia, "Times New Roman", serif;
}

Laying out web pages with CSS

Layout is one of the most challenging things to do in CSS because it requires you to think about the whole picture and plan ahead.

To creating a standard 2-column layout requires us to give dimensions to some of our content, and position it accordingly.

Generic Container Tags

Sometimes content you might want to group together (and treat differently with CSS) shouldn't be defined as a h1 or h2 or even a p paragraph. There are two kinds of generic containers available:

  • span - An inline container.
    Good for highlighting keywords like I did in this bullet.
  • div - An block container.
    Much like a paragraph, it creates a line break above and below to create a box.
<span>I'm a span!</span>
<span>I'm another span!</span>
I'm a span! I'm another span! 

<div>I'm a div!</div>
<div>I'm another div!</div>
I'm a div!
I'm another div!

Assigning widths

It is important to know the overall width of your site so you can select column widths for the main <section> and the <aside> sidebar that will make sense.

Let's go ahead and set the <header> and <footer> to be 960 pixels wide. We'll add a light gray background colour too, to make it easier to see the size of the elements.

header {
	width: 960px;
	background-color: #eee;
}

footer {
	width: 960px;
	background-color: #eee;
}

Assigning widths

If our site is 960px wide, we need to ensure that our columns don't exceed 960 - including margins! (e.g. 640 + 20 + 300 = 960)

section {
	width: 640px;
	padding: 30px;
	background-color: pink;
}

aside {
	margin-left: 20px;
	width: 300px;
	background-color: red;
}

What about the padding?

PRO TIP: For many, padding has wrecked havoc on our math, since it increases the width of the element. To avoid headaches, use the box-sizing property to make sure your math adds up.

/* apply a natural box layout model to all elements */
* { 
	-moz-box-sizing: border-box; 
	-webkit-box-sizing: border-box; 
	box-sizing: border-box; 
}

Getting columns to be side-by-side

Block-style HTML containers like <p>, <div>, <section>, and <aside> automatically start on a new line.

To get our two columns to be side-by-side, use the float property. Values can be left, right, or none.

section {
	width: 640px;
	padding: 30px;
	background-color: pink;
	float: left;
}

aside {
	margin-left: 20px;
	width: 300px;
	background-color: red;
	float: left;
}

Clearing floats

You may notice that floats have the ability to affect the rest of the web page in unexpected ways. It's important to always clear your floats.

One way is to use the clear property on an element that comes directly after the containers that are floating. Values can be left, right, or both.

footer {
	width: 960px;
	background-color: #eee;
	clear: both;
}

If the elements that are floating are enclosed in another container, you can you apply overflow:auto or overflow:hidden; to the parent container:

nav {
	overflow: auto;
}

CSS id's

CSS classes are great when you want to re-apply the same style to multiple HTML elements but sometimes you need to only target an HTML container once. This is where an ID is useful.

  • Use a hash (#) to define CSS selectors that are IDs.
    e.g. #centered
  • Use the id attribute to apply the styles to that element.
    e.g. id="centered"

Important! You should only have one instance of an ID per page. Any more can cause unexpected things to happen with your CSS -- and later, JavaScript.

CSS id's

Let's add #centered to styles.css and apply the id to the div container immediately after the opening <body> tag in index.html.

#centered {
	margin: 0 auto;
	width: 960px;
}

	</div>

Getting your website online

Using FTP

Déjà vu: What is a website?

Remember how a website is essentially a folder located on a dedicated computer (a server)?

Using FTP to put your site online

If you have access to web hosting and would like to put your site online, there are a few vital pieces of information you need:

  1. FTP Address
  2. Username
  3. Password
  4. Any special security info
  5. Which directory to transfer your files to

Where to get this information

This information will generaly be provided to you by your hosting service provider.

There ARE conventions regarding what directory your files go into once you have connected to your server via FTP:

  • www
  • public
  • public_html
  • htdocs

Transferring your files

Once you have identified where your files need to go, it is often just a matter of dragging and dropping into your FTP program.

Once this is done, you can check your web address to see how things are looking!
