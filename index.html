Organizing your files

Websites, webpages, and file structures

What is a website?

A website is made up of many webpages.

What is a website?

These webpages closely match a file folder structure.

 

What is a website?

Essentially this folder is located on a dedicated computer (also known as a server) that hosts websites.

Good file naming habits

  • Keep your file names in all lowercase.
    about.html instead of About.html or ABOUT.html
  • Avoid s p a c e s in your file name. Use underscores instead.
    business_hours.html instead of business hours.html
    (The browswer will interpret it as business%20hours.html)
  • For the same reason, avoid symboles like # & * in your file name.
  • Keep your file names concise, yet meaningful.
    about.html, contact.html instead of page1.html, page2.html

Dive into HTML

The content layer

What is HTML?

HTML was created for web browsers to easily read (not humans).

So this makes perfect sense to a web browser:

<!DOCTYPE html><html><head><title></title></head><body></body></html>

What is HTML?

Whitespace through indentation (tabs) make your more HTML human-readable.

<!DOCTYPE html>
<html>
  <head>
    <title> </title>
  </head>
  <body>
  </body>
</html>

HTML tags

The structure of a webpage is defined by HTML tags. Angled brackets denote tags, like this:

<tag>

Think about HTML tags like the beams of a house.
It's what holds up a webpage.

HTML tags

Tags typically come in pairs -- an opening tag and a closing tag with a forward slash:

<tag></tag>

<!DOCTYPE html>
<html>
  <head>
    <title> </title>
  </head>
  <body>
  </body>
</html>
<!DOCTYPE html>
<html>
  <head>
    <title> </title>
  </head>
  <body>
  </body>

HTML tags

Tags can be "nested" within other tags, like this:

<tag><tag></tag></tag>

<!DOCTYPE html>
<html>
  <head>
    <title> </title>
  </head>
  <body>
  </body>
</html>
<!DOCTYPE html>
<html>
  <head>
    <title> </title>
  </head>
  <body>
  </body>
</html>

(This relationship is often referred to using the terms "parent" and "child".)

Going deeper

Get to know HTML even better

Self-closing HTML Tags

Self-closing tags don't format content, but instead they are used to render or embed objects in our page.

One of the easiest examples of this is the <hr /> tag used to display a horizontal rule on the page.

However the most often used example of this is the <br /> tag used to create a line break.

<p>This text<br />goes over<br />several<br />lines.</p>

This text
goes over
several
lines.

Tags with attributes

HTML tags identify structure and content to the browser but sometimes we need to define data that we don't see right away but it's used by the browser to help render special content like hyperlinks, images, or video.

For example, the <a> anchor tag is used to tell the browser that we want to link to somewhere else on our site or on the web. The content between the tags is what is displayed as the link text (typically underlined).

Visit <a>Ladies Learning Code</a>.

Visit Ladies Learning Code.

Tags with attributes

We need a way of setting the URL (e.g. http://ladieslearningcode.com) that the browser will request when someone clicks on the link.

For this, we use attributes.

Visit <a href="http://ladieslearningcode.com">Ladies Learning Code</a>.

Visit Ladies Learning Code.

  • href is the attribute
  • http://ladieslearningcode.com is the value of that attribute

Working with images

Images are an example of tags with attributes and is self-closing.

<img src="llc.gif" />

The src attribute tells the browser what the url source of our image is.

(View original image.)

Working with images

Tags can have more than one attribute, some required (like src) and others are optional:

  • width and height tell the browser what width/height to display the image with (which isn't necessarily the image's actual width/hegiht)
  • alt is alternative text to display to visually impaired web visitors using screen readers.
    (Technically, it's required to be "valid" HTML markup which is why you might see alt="" often used.)
<img src="llc.gif" width="328" height="150" 
  alt="Ladies Learning Code t-shirt logo" />

(View original image.)

Working with images

In the exercises/2_WORKING_WITH_IMAGES folder, there's a img folder. It's best practice to organize your images into its own folder.

Since the image is not in the same folder as index.html, if you wanted to refer to the image you would need to use a relative path.

<img src="img/llc.gif" />

The opposite of this (going up a folder level) would be: src="../img/llc.gif"

Working with images

Useful for your reference:

  • gif - smaller file size when used for line art, charts, and images with minimal colour variation
  • jpg - smaller file size when used for photographs with lots of rich colours or gradients
  • png - larger file size but has best quality and is great for images with transparent backgrounds

CSS basics

The presentation layer

Why use CSS?

  • Seperates content (text) from presentation (text colour).
  • The "cascading" part of CSS means that it (should) help avoid redundancy in styling your web documents.

Back in the day...

<b><font color="red">Red</font></b>
<b><font color="green">Green</font></b> 
<b><font color="blue">Blue</font></b>
Red Green Blue

Re-written (and extended) as CSS:

<style>
  .red { color: red; }
  .green { color: green; }
  .blue { color: blue; }
  #colours span { font-weight: bold; padding: 5px 10px; border: 1px solid #ccc; }
  #colours span:hover { text-decoration: underline; }
</style>

<div id="colours">
  <span style="red">Red</font>
  <span style="green">Green</font>
  <span style="blue">Blue</font>
</div>
Red Green Blue

Changing the background color

Let's jump right in by changing the background colour of our web page! Try adding this into your exercises/3_CSS/index.html page between the style tag.

body { background: gray; }

Notice anything different?

CSS selectors and properties

Here's a CSS declaration dissected:

  • A basic selector will match its HTML tag name. In this case, <h1>. (We'll get into more complex ones later.)
  • A property is a special label for some visual aspect of the selector. You can see a list of them in this cheatsheet.
  • The value of the property is specific to the kind of property it is. color properties accept colour values like blue or #FFFF00. font-size properties accept units of measurements like 12px (where "px" stands for pixels).
  • Use curly braces to hold all the property:value sets.
  • You can define multiple property:value sets but you must end each one with a semi-colon.

Fonts and color

Some of you may find that your text is now a little hard to read.

  • Try changing the colors of the following tags: <h2> <p> <a>
  • What about change the colour for all text on the page?
    (This is where the term "cascading" for CSS is descriptive.)

Font family

The web has a number of fonts that are considered "web safe".

Some of the most used fonts are:

You can use these and various other fonts without having to worry about someone not being able to see them.

Font family

Let's declare a font-family for the entire document.

body { font-family: Arial, Helvetica, sans-serif; }

We are setting up Arial as our primary font with Helvetica as the first backup.

The default sans-serif font on your computer is our final backup.

  • The headings could pop more if they were a serif font. Change the font-family of <h1> and <h2>.

Dividing Your Content

HTML has a neat little tag called the <div>.

On its own, this tag does nothing. Its purpose is to allow us to categorize our HTML and, later on, target specific areas of our code.

For now, let's try and organize our content a little better.

Let's Change Our Main Section Background!

Yeah! Let's change more backgrounds!

div { background: white; }

Making Our <div> Unique

Adding an ID makes our <div> unique.

You should only have one instance of an ID per page. Any more can cause errors in your page and CSS.

Think of an ID as something unique - similar to a driver's licence -

<div id="nav">

Margins and Padding

Creating a multi-column layout

Layout is one of the most challenging things to do in CSS, but you can get predictable results in a few steps.

This requires us to give dimensions to some of our content, and position it accordingly.

The first thing we want to do is identify which content is going to become our columns, and make sure we have a way to target them in css.

This can be done by adding a meaningful ID or Class attribute.

Assigning widths

The next thing we want to do is assign a width to our columns.

It is important to know the overall width of your site so you can select column widths that will make sense -

If our site is 960px wide, then we need to ensure that our columns don't add up to a width greater than 960 - INCLUDING their padding!

We can set this using our CSS width property:

        #sidebar{
        width: 300px;
        padding: 20px 30px 20px 0;
        }

Getting Columns to Split

Once we have our widths in place, we can get our columns to split to where they need to go -

This can be done by taking advantage of our float property, and setting our left column to float: left; and our right column to float: right;. 

        .sidebar{
        width: 300px;
        padding: 20px 30px 20px 0;
        float: right;
        }

CSS Classes

We can also group some elements together and edit them all at the same time.

Similar to how we can assign ID's to content using an attribute, we can do the same thing with a class.

In order to give styles to our nagivation's links we need to assign them a class.

<img class="thumbnail" src="..." />

And to target it in CSS we use a "."

.thumbnail { float: right; }

Unlike the ID, we can use a class however many times we want.

Some Basic Interaction

CSS is also able to handle some basic interaction in our sites, such as allowing a 'hover' state on various elements.

A hover state is when a site user hovers over an element with their mouse.

This is frequently done with items such as links.

In CSS, you are able to change the CSS properties when somebody hovers over an item as follows:

        .btn:hover{
        text-decoration: underline;
        color: #777777;
        }

Getting your website online

Using FTP

Using FTP to put your site online

If you have access to web hosting and would like to put your site online, there are a few vital pieces of information you need:

  1. FTP Address
  2. Username
  3. Password
  4. Any special security info
  5. Which directory to transfer your files to

Where to get this information

This information will generaly be provided to you by your hosting service provider.

There ARE conventions regarding what directory your files go into once you have connected to your server via FTP:

  • www
  • public
  • public_html
  • htdocs

Transferring your files

Once you have identified where your files need to go, it is often just a matter of dragging and dropping into your FTP program.

Once this is done, you can check your web address to see how things are looking!
